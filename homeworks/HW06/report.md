## 1. Dataset

**Выбранный датасет:** `S06-hw-dataset-04.csv`

**Размер:** 25000 строк × 62 столбца

**Целевая переменная:** `target` (бинарная классификация: 0 и 1)
- Класс 0 (большинство): 95.08%
- Класс 1 (меньшинство): 4.92%

**Признаки:** 
- 60 числовых признаков (`f01` – `f60`)
- Столбец `id` (не используется как признак)
- Пропусков нет
- Все признаки непрерывные (тип: float64)

**Особенности:** 
В данных наблюдается **выраженный дисбаланс классов** с пропорцией приблизительно 95:5, что характерно для задач поиска аномалий (обнаружение мошенничества, редкие события). Этот фактор крайне важно учитывать при подборе оценочных метрик — accuracy может быть недостоверной, поскольку модель способна достичь 95% точности, просто всегда предсказывая мажоритарный класс.

## 2. Protocol

### Разбиение данных
- **Train/Test split:** 75% / 25% (`test_size=0.25`)
  - Train: 18750 объектов
  - Test: 6250 объектов
- **Стратификация:** применена (`stratify=y`) для сохранения пропорций классов
  - Train: класс 0 = 95.08%, класс 1 = 4.92%
  - Test: класс 0 = 95.09%, класс 1 = 4.91%
- **Random state:** 42 (для воспроизводимости)

**Обоснование:** Стратификация критична при дисбалансе — гарантирует, что в train и test будут одинаковые доли классов, что особенно важно для редкого класса 1, который составляет менее 5%.

### Подбор гиперпараметров
- **Метод:** GridSearchCV
- **Cross-Validation:** 5-fold CV на train
- **Оптимизируемая метрика:** `roc_auc` (лучше подходит для дисбаланса, чем accuracy)
- **Важно:** Все эксперименты с гиперпараметрами проводились **только на train**. Test использован **один раз** для финальной оценки.

### Метрики качества
1. **Accuracy** — доля правильных предсказаний (базовая метрика, но обманчива при дисбалансе)
2. **F1-score** — гармоническое среднее precision и recall (учитывает оба типа ошибок)
3. **ROC-AUC** — способность модели разделять классы при разных порогах (основная метрика для выбора модели)
4. **PR-AUC** (Average Precision) — дополнительная метрика, более информативная для дисбаланса

**Почему эти метрики уместны:**
- При дисбалансе 95/5 модель может достичь accuracy=95%, просто предсказывая всегда класс 0
- F1 и ROC-AUC показывают реальную способность модели различать классы
- PR-AUC особенно важна для дисбаланса, так как фокусируется на качестве предсказаний для редкого класса

Высокое значение PR-AUC (0.7647) также подтверждает, что модель эффективно работает именно с редким классом 1.

## 5. Analysis

Проверка стабильности модели не выполнялась (необязательный этап). При заданном фиксированном значении random_state=42 результаты являются абсолютно воспроизводимыми. Все промежуточные результаты и метрики сохранены, что позволяет повторить эксперимент.

### Анализ ошибок

**Classification Report для RandomForest:**

```
              precision    recall  f1-score   support

     Class 0       0.97      1.00      0.98      5943
     Class 1       0.99      0.40      0.57       307

    accuracy                           0.97      6250
   macro avg       0.98      0.70      0.78      6250
weighted avg       0.97      0.97      0.96      6250
```

**Интерпретация:**

**Класс 0 (большинство, 5943 объекта):**
- Precision = 0.97 — из предсказанных "0" 97% реально были "0"
- Recall = 1.00 — модель нашла практически все случаи класса 0
- F1 = 0.98 — отличное качество для класса большинства

**Класс 1 (меньшинство, 307 объектов):**
- Precision = 0.99 — из предсказанных "1" 99% реально были "1" (почти нет ложных тревог!)
- Recall = 0.40 — модель нашла только 40% случаев класса 1 (пропустила 60%)
- F1 = 0.57 — умеренное качество из-за низкого recall

**Ключевые наблюдения:**

1. **Trade-off между precision и recall:** Модель выбрала стратегию "высокая precision, низкий recall" для класса 1. Это означает, что когда модель предсказывает класс 1, она почти всегда права (99%), но она делает это редко, пропуская 60% случаев.

2. **Для fraud detection** такой баланс не идеален — лучше иметь чуть больше ложных тревог (lower precision), но находить больше реальных случаев (higher recall). Это можно улучшить через:
   - Настройку порога классификации (threshold tuning)
   - Использование `class_weight='balanced'`
   - Применение SMOTE для балансировки классов

3. **Дисбаланс сильно влияет:** При 95:5 модель "осторожничает" с предсказанием редкого класса. Высокий macro avg recall (0.70) скрывает проблему низкого recall для класса 1.

## 6. Conclusion

**Основные выводы:**

1. Отдельные деревья склонны к переобучению, ансамбли решают эту проблему: DecisionTree продемонстрировал ROC-AUC = 0.7948, что существенно ниже показателей ансамблей (Random Forest = 0.9016, GradientBoosting = 0.8960). Это доказывает, что одиночные деревья имеют высокую дисперсию, а методы bagging или boosting в ансамблях успешно её сокращают. Random Forest улучшил ROC-AUC на +10.7 процентных пункта по сравнению с одним деревом.

2. Сравнение Random Forest и Boosting: неожиданно выиграл bagging: Несмотря на то, что boosting обычно превосходит bagging, в данном случае Random Forest (0.9016) показал более высокий ROC-AUC, чем Gradient Boosting (0.8960). Возможное объяснение — на этих данных случайный выбор признаков (Random Forest) оказался эффективнее последовательного исправления ошибок (Boosting), вероятно, из-за наличия множества слабоскоррелированных признаков.

3. Accuracy может вводить в заблуждение при дисбалансе классов: Важный вывод: модели с accuracy 0.9786 (GB, Stacking) уступили модели с accuracy 0.9771 (RF) по ROC-AUC. При соотношении классов 95:5 высокая точность часто достигается за счёт корректного предсказания доминирующего класса, но это не говорит о лучшей способности различать классы. Для выбора модели правильнее использовать метрики F1, ROC-AUC и PR-AUC.

4. Строгий ML-протокол важен для достоверности результатов: Объективная оценка была обеспечена подбором гиперпараметров через 5-кратную кросс-валидацию на обучающей выборке с оптимизацией по ROC-AUC и однократной проверкой на тестовом наборе. Фиксация random_state=42 на всех этапах (разбиение, модели, CV) обеспечила полную воспроизводимость. Все артефакты (модели, метрики, графики) сохранены для проверки эксперимента.

5. Интерпретируемость на основе permutation importance: Анализ значимости признаков выявил, что модель использует разнообразные данные (топ-15 признаков суммарно дают ~11% важности), а не полагается на 1-2 ключевых. Это повышает устойчивость модели к изменениям в данных. Признаки f54, f41, f13 являются наиболее значимыми для разделения классов и важны для интерпретации предсказаний.

6. Компромисс Precision-Recall для задачи обнаружения мошенничества: RandomForest достиг precision=0.99 для класса 1, но recall=0.40, что означает, что 60% случаев пропускаются. Для промышленной системы обнаружения необходима дополнительная настройка: подбор порога для увеличения recall, использование class_weight='balanced' или методов балансировки (например, SMOTE). Текущая модель подходит для задач, где критически важна точность детекции, а пропуск части случаев допустим.
